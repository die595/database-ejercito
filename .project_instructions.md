# DATACORE INTEL - Project Instructions

## Overview
Criminal intelligence analysis dashboard for Colombia. Supports 500k+ records.

## Authentication System
- Custom JWT auth (jose library) with bcryptjs passwords
- Cookie: `datacore_session` (httpOnly, 7-day expiry)
- First registered user → admin + approved automatically
- Subsequent users → user + pending (need admin approval)
- Admin panel accessible from header "USUARIOS" button
- Middleware protects all routes except: /login, /register, /api/auth/*
- Requires `app_users` table in Supabase (user must create via SQL)

## Architecture (Post-Supabase Migration)
- **Data Storage**: Supabase (external) - table `intel_records` with 23 columns
- **Flow**: Excel/CSV → Server parses → Inserts into Supabase → Dashboard queries Supabase with filters → Only aggregated/sampled data reaches browser
- **The browser NEVER holds all records in memory**

## Supabase Config
- URL: `NEXT_PUBLIC_SUPABASE_URL` in .env
- Anon key for reads, Service role key for writes/deletes
- Table: `intel_records` (BIGSERIAL PK, text cols, DOUBLE PRECISION lat/lon, DATE fecha)
- Indexes on: departamento, municipio, tipologia, fecha, estructura, fenomeno_criminalidad, (latitud,longitud)

## API Routes
- `/api/intel/import` - POST: Excel/CSV upload → parse → batch insert Supabase (NDJSON progress stream)
- `/api/intel/stats` - GET: Paginated aggregation (5k/page, 8 parallel) → returns all stats + analysis + municipiosByDept
- `/api/intel/map-points` - GET: Paginated map coordinates (max 15k sampled)
- `/api/intel/filters` - GET: Distinct values for dropdowns (legacy, not actively used)
- `/api/intel/destroy` - DELETE: Remove all records

## Context (intel-context.tsx)
- No in-memory records. State: isDataLoaded, isLoading, filters, stats, mapPoints, uniqueValues
- On mount: quick Supabase count check → if data exists, load stats + map
- Filter changes trigger API refetch of stats + map-points
- uniqueValues derived from initial unfiltered stats (stored once)

## Visual Style
- Modern dark-slate theme: bg-gradient-to-br from-slate-900 via-slate-800 to-slate-900
- Cyan accents, slate borders/cards
- Font: DM Sans display, monospace for data
- Leaflet normal tiles (OpenStreetMap) with white popups
- Canvas renderer for map performance
- Tooltips: slate-800 bg, light text, good contrast

## Role-Based Access
- Admin: can import Excel, destroy data, add records via grid, manage users
- Normal user: read-only dashboard, no import/destroy/grid-edit access
- DataGrid at bottom of dashboard: admin-only editable, locked for normal users

## Key Technical Decisions
- ExcelJS streaming reader for large Excel files (SheetJS fails on 98MB)
- Stats route paginates through all records server-side, aggregates incrementally
- Map points sampled to max 15k for performance
- PDF generated client-side with jsPDF + html2canvas

## Language
- User speaks Spanish. All UI and responses in Spanish.
